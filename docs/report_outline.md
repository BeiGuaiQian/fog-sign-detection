# 基于图像去雾与目标定位的雾天路标检测系统报告大纲

## 1. 摘要

- 雾天会降低交通图像的对比度、清晰度和目标可见性，导致路标检测难度上升。
- 本项目构建了一个“图像去雾 + 目标定位 + 指标评价 + 可视化展示”的完整系统。
- 方法上，先使用 DCP 暗通道先验算法对雾天图像进行去雾，再使用 YOLOv8 对原始雾图和去雾图分别进行路标检测。
- 结果上，从图像质量指标和检测指标两方面比较去雾前后的变化，包括亮度、对比度、信息熵、拉普拉斯清晰度、边缘数量、检测框数量和平均置信度。
- 系统价值：验证传统图像增强方法对深度学习目标检测的辅助作用，并提供可交互的实验平台。

## 2. 研究背景与意义

### 2.1 雾天图像退化问题

- 雾、霾等天气会使大气散射增强，导致图像整体发白、对比度下降、颜色失真、边缘模糊。
- 对交通场景而言，路标通常面积较小，雾天退化会进一步降低其纹理、边缘和颜色辨识度。
- 图像退化不仅影响人工观察，也会影响目标检测模型的特征提取和定位精度。

### 2.2 路标检测在交通视觉中的意义

- 交通路标检测是智能驾驶、辅助驾驶、交通监控等视觉系统的重要任务。
- 路标包含限速、警告、禁止、指示等关键信息，检测结果会影响后续决策。
- 在复杂天气下保持路标检测稳定性，是交通视觉系统鲁棒性研究的重要方向。

### 2.3 图像去雾对目标检测的辅助作用

- 去雾可以提升图像局部对比度和边缘清晰度，使目标轮廓更明显。
- 对 YOLOv8 等检测模型而言，更清晰的图像可能提高检测框数量和检测置信度。
- 本项目通过“原图检测 vs 去雾后检测”的实验方式，验证去雾处理对路标定位效果的影响。

## 3. 相关理论

### 3.1 大气散射模型

- 雾天图像通常可用大气散射模型表示：

```text
I(x)=J(x)t(x)+A(1-t(x))
```

- 其中，`I(x)` 表示观测到的有雾图像，`J(x)` 表示待恢复的无雾场景辐射，`A` 表示全局大气光，`t(x)` 表示透射率。
- 当透射率较低时，目标真实信息被大气光遮蔽，图像呈现发白和模糊。

### 3.2 暗通道先验 DCP

- 暗通道先验认为，在大多数非天空自然图像局部区域中，至少有一个颜色通道存在接近 0 的像素值。
- 对有雾图像而言，暗通道会被大气光抬高，因此可以利用暗通道估计雾的浓度。
- 本项目通过对 RGB/BGR 三通道取最小值，并进行最小值滤波得到暗通道图。

### 3.3 大气光估计

- 大气光 `A` 用于描述雾天环境中散射光的颜色和强度。
- 本项目从暗通道值最大的前若干像素中，选择原图三通道亮度和最大的像素作为大气光估计值。
- 大气光估计会影响最终图像恢复效果，估计过高或过低都可能造成颜色偏移。

### 3.4 透射率估计

- 透射率反映场景点到相机之间光线未被散射的比例。
- 本项目采用 DCP 公式估计初始透射率：

```text
t(x)=1-omega*dark_channel(I/A)
```

- `omega` 用于保留少量雾感，避免过度增强造成图像不自然。
- 为避免恢复时噪声被过度放大，设置最小透射率阈值 `t0`。

### 3.5 引导滤波

- 初始透射率图可能存在块效应和边缘不连续问题。
- 引导滤波利用灰度图作为引导图，对透射率图进行边缘保持平滑。
- 本项目使用 OpenCV 的 `boxFilter` 实现引导滤波，提高去雾结果的自然性。

### 3.6 YOLOv8 目标检测基本思想

- YOLO 系列是一类单阶段目标检测方法，直接从图像中预测目标类别、检测框位置和置信度。
- YOLOv8 具有较好的速度和精度平衡，适合课程项目中的实时可视化检测。
- 本项目使用 Ultralytics YOLOv8 对原始雾图和 DCP 去雾图分别推理，输出检测框、类别名称和置信度。

## 4. 系统设计

### 4.1 总体流程

- 用户上传雾天路标图片。
- 系统将图片转换为 OpenCV BGR 格式。
- DCP 模块输出去雾图、暗通道图和透射率图。
- YOLOv8 模块分别检测原图和去雾图。
- 指标模块计算图像质量和检测结果统计。
- Streamlit 页面展示图像、表格和下载按钮。

【插入图：系统流程图】

### 4.2 图像去雾模块

- 输入：OpenCV BGR 图像。
- 输出：去雾 BGR 图像、暗通道图、透射率图、调试参数。
- 主要步骤：暗通道计算、大气光估计、透射率估计、引导滤波、场景辐射恢复。
- 对应代码文件：`src/dehaze_dcp.py`。

### 4.3 路标检测模块

- 输入：原始雾图或去雾图。
- 模型：Ultralytics YOLOv8。
- 输出：带检测框的图像，以及包含类别、置信度、坐标、中心点、宽高和面积的检测列表。
- 如果 `weights/best.pt` 不存在，系统会临时使用 `yolov8n.pt`，但最终实验建议使用交通路标专用权重。
- 对应代码文件：`src/detect_yolo.py`。

### 4.4 指标评价模块

- 图像质量指标：`brightness_mean`、`contrast_std`、`entropy`、`laplacian_sharpness`、`edge_pixels`、`edge_ratio`。
- 检测指标：`detection_count`、`mean_confidence`、`max_confidence`、`total_area`、`mean_area`。
- 对比指标：平均置信度提升、检测数量变化。
- 对应代码文件：`src/metrics.py`。

### 4.5 Streamlit 可视化模块

- 支持图片上传、参数调节、图像展示、检测结果表格、指标对比表和结果下载。
- 页面展示内容包括原图、去雾图、暗通道图、透射率图、原图检测结果、去雾后检测结果。
- 对应代码文件：`app.py`。

## 5. 实现方法

### 5.1 Python + OpenCV 实现 DCP

- 使用 NumPy 进行图像数组计算。
- 使用 OpenCV 的 `erode` 实现暗通道局部最小值滤波。
- 使用 OpenCV 的 `boxFilter` 实现引导滤波。
- 使用大气散射模型反解无雾图像。
- 输出中间结果用于报告分析和页面可视化。

### 5.2 Ultralytics YOLOv8 实现路标定位

- 通过 `TrafficSignDetector` 类封装模型加载和推理。
- 使用 `model.predict` 对图像进行检测。
- 检测结果整理为列表字典，便于保存为 CSV 或显示为表格。
- 标注图使用 OpenCV 绘制，保证 BGR 格式可直接保存。

### 5.3 Pandas 统计检测结果

- 将检测框信息转换为 `DataFrame`。
- 将单张图片的对比指标保存为扁平字典。
- 批量实验中，将多张图片的结果汇总为 `summary.csv`。
- 进一步计算所有数值指标平均值，保存为 `summary_mean.csv`。

### 5.4 Streamlit 实现交互界面

- 使用侧边栏设置模型权重、设备、YOLO 阈值和 DCP 参数。
- 使用 `file_uploader` 上传图片。
- 使用多列布局展示去雾前后图像和检测结果。
- 使用 `dataframe` 展示检测框与指标表。
- 使用 `download_button` 下载去雾图和检测结果图。

### 6.3 对比方法

- 对同一张雾天图像分别执行两条检测路径：
  - 原始雾图 → YOLOv8 检测。
  - 原始雾图 → DCP 去雾 → YOLOv8 检测。
- 比较两条路径的检测框数量、平均置信度和最高置信度。
- 同时比较原图和去雾图的图像质量指标。

### 6.4 图像质量指标

- `brightness_mean`：灰度均值，反映整体亮度。
- `contrast_std`：灰度标准差，反映图像对比度。
- `entropy`：信息熵，反映图像信息量。
- `laplacian_sharpness`：拉普拉斯方差，反映边缘和清晰度。
- `edge_pixels`：Canny 边缘像素数量，反映边缘结构数量。

### 6.5 检测指标

- `detection_count`：检测框数量。
- `mean_confidence`：平均检测置信度。
- `max_confidence`：最高检测置信度。
- 可结合检测框面积分析去雾是否使小目标更容易被定位。

### 7.4 去雾对检测效果的影响分析

- 如果去雾后检测数量增加，说明增强后的边缘和纹理可能帮助模型识别路标。
- 如果平均置信度提高，说明去雾图像可能更接近模型训练时的清晰图像分布。
- 如果图像质量指标提高但检测效果没有明显提升，可能说明 YOLOv8 对该类路标或该雾化程度仍不敏感。
- 需要结合图像和检测框，不只依赖单一指标判断效果。

### 7.5 失败案例分析

- DCP 在天空区域、强光区域或大面积白色区域可能出现大气光估计偏差。
- 去雾后可能出现颜色偏移、过度增强或噪声放大，影响检测稳定性。
- 极重雾场景中，路标本身信息缺失严重，单幅图像去雾难以完全恢复目标细节。
- 如果使用 `yolov8n.pt` 临时模型，由于它不是路标专用模型，检测结果可能无法代表真实路标检测能力。

## 8. 总结与展望

### 8.1 已完成内容

- 实现了基于 DCP 暗通道先验的单幅图像去雾算法。
- 封装了 YOLOv8 路标检测模块。
- 构建了图像质量指标和检测指标评价模块。
- 实现了 Streamlit 可交互系统，支持上传、处理、展示和下载。
- 实现了单图测试脚本和批量实验脚本，支撑课程报告中的实验分析。

### 8.2 不足

- DCP 依赖暗通道先验，在天空、白色物体和强光区域可能效果不稳定。
- DCP 去雾可能导致颜色偏移、局部过增强或噪声放大。
- YOLO 对极重雾、远距离小路标和遮挡路标仍可能漏检。
- 如果没有交通路标专用权重，通用 `yolov8n.pt` 的实验效果有限。

### 8.3 展望

- 引入 AOD-Net、DehazeFormer 等学习型去雾方法，与 DCP 进行对比。
- 使用更大规模的多天气交通路标数据集训练 YOLOv8，提高模型鲁棒性。
- 加入视频序列处理和目标跟踪，提高连续场景中的检测稳定性。
- 将去雾参数自适应化，根据图像雾浓度自动调整 `omega`、`t0` 等参数。
- 增加人工标注真值，计算 mAP、Precision、Recall 等更完整的检测评价指标。
